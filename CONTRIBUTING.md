# 贡献指南

这个项目欢迎真实案例、方法改进、平台适配和新的验收场景。

## 什么值得贡献

- 一个经过真实用户验证的创业动作。
- 一段从内容到咨询、报价、成交或复购的完整数据。
- 一个失败案例，以及最早出现的错误信号。
- 一个更清楚、更短、更容易行动的流程。
- 一个新的合规风险或隐私检查项。

## 不接受什么

- “一定赚钱”“一键爆款”等无法验证的承诺。
- 只有观点，没有用户、动作和结果的空泛方法论。
- 客户、供应商、账号、密码、邀请链接和未经授权的聊天。
- 绕过审核、刷量、欺诈、学术作弊和侵权方法。
- 从其他受限 Skill 或付费资料中直接复制的内容。

## 提交格式

每次贡献尽量回答：

1. 用户是谁，遇到了什么问题？
2. 原假设是什么？
3. 做了什么动作？
4. 得到了什么真实证据？
5. 收入、成本、工时和风险是什么？
6. 这条经验应该修改哪个文件？
7. 应新增哪个验收场景？
