# TC v1.7.0｜接入 DBS Markdown 外部理论库

## 这次解决什么

过去 TC 只参考 DBS 的 Skill 架构，不能直接按问题检索 dontbesilent 发布的开源推文集。v1.7.0 新增一套真实可运行、同时守住许可证边界的外部理论库能力。

## 新能力

- 支持从原作者 GitHub 仓库同步 `dontbesilent-开源推文集.md`；
- 只使用适合 AI 读取的 Markdown，不读取 PDF；
- 解析日期、内容类型、正文、原帖链接、主题、表达方式和标签；
- 检索时优先匹配标签与主题，再匹配正文；
- 返回结果保留日期、原链接、第三方观点和证据边界；
- 支持本地文件、目录、环境变量和默认缓存目录四种来源；
- 已用上游真实文件验证可解析 1,544 条记录，日期范围为 2024-10-01 至 2026-07-19。

## 许可边界

DBS 仓库采用 CC BY-NC 4.0，TC 自身采用 Apache-2.0。为避免混淆：

- 上游全文不进入 TC 仓库和安装包；
- 用户必须明确接受署名、非商业限制后再同步；
- Markdown 直接从原作者仓库下载到用户本机；
- 商业文案、课程、付费交付和产品不得复制或改写上游原文，除非另行取得作者许可；
- 外部观点不自动升级成 TC 事实或稳定方法。

## 怎样使用

第一次同步：

```bash
python3 scripts/tc_knowledge.py external-sync \
  --source dbs-books --accept-license
```

按问题查找：

```bash
python3 scripts/tc_knowledge.py search \
  --scope dbs-books --query "流量 变现 产品" --limit 5
```

普通用户仍然只需要输入 `/tc`。明确要求结合 DBS 开源推文集时，TC 会先定义当前真问题，再只查最相关的 3 至 5 条记录。
