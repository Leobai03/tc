# TC v1.6.0｜核心参考源与更清楚的 GitHub 入口

## 这次解决什么

过去技术合伙人的知识付费与实操记录只被零散提炼成几条 AI 原则，没有真正成为 TC 的主要判断来源；同时 Release 页平铺了全部 Skill 压缩包，普通用户容易误以为整个项目就是一堆 ZIP。

v1.6.0 同时修正知识结构和使用入口。

## 核心参考源

- 新增 `知识库/核心参考源/`；
- 将技术合伙人的商业与 AI 实操体系提升为核心参考源 A，优先用于商业化、需求、产品、流量、AI 与 To B 交付；
- 将天策公开实践与创业价值观整理为核心参考源 B，优先用于行动、内容、个人 IP、连接、社群与公开成长；
- 明确“核心参考源”是高频使用的判断视角，不是更高等级的事实证据；
- 技术合伙人的私有原文、链接、身份、客户、收入和经营细节不进入公开仓库；
- 明确剔除绝对化、操控性、未经核验和可能伤害用户的做法。

## 知识检索升级

- `/tc` 会先按问题选择一份核心参考源，再选择一个专项知识包落到行动；
- `/tc-knowledge` 新增 `sources` 搜索范围；
- 默认搜索可以同时返回核心参考源、知识包、知识原子和历史原推；
- 统计与校验会检查 2 份核心参考源是否存在、脱敏且同步进安装包。

## ZIP 入口修正

- README 顶部直接解释 Code 页面与 Release 页的区别；
- 想看 DBS 式文件夹结构，应进入仓库 Code 首页；
- 普通用户仍优先使用一条安装命令，不需要下载 ZIP；
- 正式 Release 只展示 `tc-suite-x.y.z.zip` 和 `tc.zip` 两个安装包；
- 各子 Skill ZIP 继续在构建阶段生成和测试，但不再铺满 Release 下载页。

## 普通用户怎么用

推荐安装：

```bash
npx -y skills add Leobai03/tc -g --all
```

然后新开对话输入：

```text
/tc
```

明确查知识时：

```text
/tc-knowledge 用技术合伙人的商业框架，帮我看这个 AI 项目最大的未知是什么。
```

发布前检查版本与插件清单、2 份核心参考源、489 条历史公开内容、22 条知识原子、6 个知识包、知识副本同步、搜索行为、所有运行脚本，以及完整套件与主入口安装包。
