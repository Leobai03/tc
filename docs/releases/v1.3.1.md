# TC v1.3.1

这不是增加一堆新功能，而是把 v1.3.0 已经存在的能力解释清楚、打包清楚、展示清楚。

## 修复的问题

v1.3.0 的仓库有 Skill、参考资料、测试和工具，但首页缺少完整的用户视角说明；GitHub Release 正文甚至只有一条 Changelog 链接。陌生人进入仓库后，很难快速回答：TC 是什么、怎么用、这些文件夹干什么、下载哪个文件。

## 这次补了什么

- 重写 GitHub 首页，用一句人话解释 TC；
- 增加一段完整的两轮对话，让用户看到真实使用过程；
- 增加“5 个 Skill 分别干什么”和完整目录说明；
- 新增《新手入门》《工作原理》《真实使用示例》三份文档；
- 完整安装包同时附带 README、三份说明书、版本号和许可证；
- Release 流程改为优先读取正式版本说明，不再自动生成空正文；
- 自动校验必要文档、核心说明和对应版本的 Release 文件。

## 30 秒开始

```bash
npx -y skills add Leobai03/tc -g --all
```

安装后新开一个对话：

```text
/tc
```

然后把真实情况直接说出来，乱一点也没关系。

## 下载哪个文件

- `tc.zip`：只需要主入口时下载；
- `tc-suite-1.3.1.zip`：推荐，包含全部 5 个 Skill 和完整说明；
- 其他单独 ZIP：只下载明确需要的子 Skill。

TC 不承诺使用者一定赚钱。它帮助用户找到当前真正的问题，做出取舍，并尽快用真实市场证据验证判断。
