# TC 新手入门

这份说明只解决一件事：让一个第一次看到 TC 的人，完成安装并真正用完第一轮。

## 你只需要记住一个命令

```text
/tc
```

你不需要先学创业理论，也不用判断该选哪个 Skill。把真实情况直接说出来，乱一点也没关系。

## 第一步：安装

### 支持 Agent Skills 的工具

在终端执行：

```bash
npx -y skills add Leobai03/tc -g --all
```

安装完成后，重新打开一个对话，再输入 `/tc`。

如果工具没有立刻识别新 Skill，先彻底退出并重新打开应用。仍然不生效时，检查 Skill 是否安装在该工具实际读取的目录中。

### Claude Code 插件市场

```bash
claude plugin marketplace add Leobai03/tc
claude plugin install tc@tc-skills
```

### 没有原生 Skill 功能的聊天 AI

豆包、DeepSeek、Kimi 或微信里的普通 AI 不一定能安装完整 Skill。此时打开 [TC 统一入口](https://leobai825.icu/tc.html)，复制“轻量版提示词”，把它作为新对话的第一条消息发送。

轻量版保留核心流程，但不一定拥有完整知识库、自动路由和更新能力。

## 第二步：完成第一次使用

新对话输入：

```text
/tc
```

看到下面这句话，说明主入口已经生效：

```text
在。把事情直接发过来，乱一点也没关系。我先帮你把问题重新说清楚。
```

接下来，不要只说“我想赚钱”或“帮我创业”。尽量把下面四件事里已经知道的部分说出来：

1. 你现在在做什么；
2. 已经发生了什么结果；
3. 你最纠结什么；
4. 你不能接受什么代价。

不完整也可以。例如：

```text
我做职场自媒体三个月，有一万粉丝，每天有人私信，但没有稳定产品。
我想做咨询，又怕交付太重，也不知道大家到底愿不愿意付钱。
```

## 第三步：看懂 TC 的两轮回答

复杂问题通常不会一轮全部回答。

第一轮，TC 会做两件事：

- 把你真正卡住的地方重新说一遍；
- 只确认一个会改变答案的问题。

你只要回复：

- “对”；
- “不对，真正情况是……”；
- 或直接回答它问的那一个问题。

第二轮，TC 才会给：

- 一个明确判断；
- 一个推荐方案；
- 你需要接受的主要代价；
- 现在能做的第一步；
- 判断这一步有没有用的标准。

简单请求会直接回答。例如：

```text
/tc 客户需求已经确认，请把“明天下午三点开会”改得礼貌一点。
```

这种情况不需要重新讨论创业方向。

## 第四步：把结果带回来

TC 的价值不在第一次回答，而在“做完一点，再根据真实结果继续”。

执行后，把这些信息带回同一个对话：

- 你联系了多少人；
- 对方原话是什么；
- 有多少人回复、报价、付款或拒绝；
- 实际花了多少时间和钱；
- 哪一步和预期不一样。

然后再次输入：

```text
/tc
```

TC 会承接上一步，不要求你从头介绍。

如果准备换一个新对话，可以先说：

```text
/tc 保存当前进度，项目名叫“我的第一个付费产品”
```

TC 会明确告诉你存档写在本机哪里。下次输入：

```text
/tc 恢复“我的第一个付费产品”
```

即可接着上次的下一步继续。不要把账号密码、客户名单或未公开合同全文放进存档。

## 什么时候用子 Skill

普通用户不需要记，但你已经非常清楚任务时可以直接调用：

| 命令 | 用途 |
| --- | --- |
| `/tc-diagnosis` | 只想重构和定义当前问题 |
| `/tc-copy` | 已经确定要写推文、报价、私聊或跟进文案 |
| `/tc-action` | 已经有判断，需要马上变成行动 |
| `/tc-state` | 保存、恢复或汇总某个创业项目的进度 |
| `/tc-update` | 检查并安装官方最新版 |

## 三个常见误区

### 1. 把 TC 当算命工具

TC 可以给推荐，但推荐必须基于事实、约束和可验证结果。它不会替你保证未来。

### 2. 只收集答案，不做动作

如果连续几轮都在换问题、要更多方案、继续研究，却没有接触客户和市场，TC 应该指出这一点，而不是继续生产“看起来很厉害”的计划。

### 3. 把所有问题一次塞进来，要求一次全部解决

你可以一次讲很多，但 TC 会先选最限制结果的一件事。前一环没通，优化后一环通常没有意义。
