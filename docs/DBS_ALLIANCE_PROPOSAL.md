# 给 Don 哥的 DBS 协同提案

Don 哥，我根据自己过去的创业、自媒体和项目复盘资料，做了一个准备开源的 `TC` Skill。

它不是再做一套 DBS，也不会复制或重新分发 DBS 的内容。我的设想是：

- DBS 负责把目标、问题、对标、行动阻力和内容质量诊断清楚。
- `TC` 负责把诊断继续推进到真实用户、最小商品、获客内容、报价、付款、交付、毛利、复购和转介绍。

两边通过 Skill 名称动态路由，每轮只使用当前最必要的一个 DBS Skill。DBS 得出结论后，`TC` 会统一生成：

```text
DBS 核心结论
当前创业阶段
现在唯一卡点
24 小时市场动作
需要带回的收入证据
```

我给自己的验收标准不是“生成了一份方案”，而是逐步接近：有效对话、明确报价、定金、回款、正毛利、复购或转介绍。

`TC` 使用 Apache-2.0 开源。DBS 只作为用户独立安装的可选依赖，继续遵守 DBS 自己的 CC BY-NC 4.0，不混合授权，也不声称获得官方背书。

我想申请加入你后续的优质 Skills 互推或联盟。如果你已经有准入标准、互推协议、接口约定或许可证要求，我愿意按标准调整，也愿意持续提交真实用户案例和结果数据。
