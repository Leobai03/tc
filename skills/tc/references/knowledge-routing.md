# 外部知识源路由

TC 的内置参考资料负责“怎么判断”，用户自己的知识库负责“事实是什么”。两者必须分开。

## 什么时候读取外部知识源

出现以下情况时才读取，不要每轮把整个知识库塞进上下文：

- 用户提供了飞书、Notion、网盘、代码仓库或本地资料目录；
- 当前问题涉及用户过去做过什么、现在在做什么、真实数据、个人文风或已有项目；
- 用户明确要求“结合我的资料”“看我的历史内容”“更新知识库”；
- 工作区存在 `knowledge/SOURCE_OF_TRUTH.md`、`SOURCE_OF_TRUTH.md` 或项目级 `AGENTS.md`。

简单通用问题直接使用 TC 内置方法，不为了显示资料丰富而额外检索。

## 先分清问题类型

| 用户在问 | 首选真源 |
| --- | --- |
| 现在的业务、目标、价格、合作和行动 | 用户当前说明；其次是带更新时间的“当前状态”页 |
| 过去说过什么、内容风格、历史互动 | 带时间戳的原始公开内容 |
| 收入、成交、成本和交付 | 付款、合同、后台、交付记录和用户原话 |
| 应该怎么判断和行动 | TC 内置方法、脱敏案例与当前市场证据 |
| 平台规则、法律、价格和第三方状态 | 最新官方来源，必要时联网核对 |

## 读取顺序

1. **找导航。** 优先读项目级 `AGENTS.md` 和 `knowledge/SOURCE_OF_TRUTH.md`。没有导航时，先建立最小资料地图，不盲读全部文件。
2. **选最小资料。** 只读取能改变当前判断的一页、一节或一组记录。
3. **标记四件事。** 日期、口径、来源、状态（当前有效 / 历史事实 / 个人观点 / 待验证）。
4. **处理冲突。** 日期更新、证据更直接、口径更清楚的来源优先；仍冲突就明确写待确认。
5. **回到 TC。** 资料只负责补事实，最终仍按“真正问题 → 判断 → 取舍 → 行动 → 反馈”完成解题。

## 大量外部帖子与实操资料

用户提供某个人的大量帖子、会议记录、课程稿或行业资料，并要求吸收到知识库时，继续读取 [source-distillation.md](source-distillation.md)。

不要把整份资料直接并入主 Skill。按照以下顺序处理：

```text
原始资料（保留出处，不代表认可）
  ↓
候选知识原子（事实 / 经验 / 观点 / 风险）
  ↓
已验证模式（有场景、动作、证据和反例）
  ↓
TC 稳定方法（脱敏、通用、可执行、可降级）
```

这套结构借鉴 DBS 的渐进式内容工程：先稳定对象、来源、关系和去重规则，再扩大规模。首次处理的目标是产出一批可用样本和筛选规则，不承诺一次吃完整个资料库。

## 飞书知识库

飞书页面适合维护当前状态、人工判断、项目导航和行动记录。

- 先读取目录或大纲，再按标题局部读取；不默认拉取整个知识空间。
- 有“当前状态 / 动态事实 / 待确认项”时优先读取。
- 历史页面和旧版本保留为档案，不自动覆盖最新状态。
- 修改内容前确认目标页面和范围；写完重新读取验证。

作者天策使用私有飞书维护当前状态和内部资料，但私有页面地址不写入公开安装包。使用者需要在自己的环境中配置链接或连接器；不得声称安装 TC 后会自动获得任何飞书权限。

## X（Twitter）导出目录

X 导出适合做历史内容研究，不适合直接回答今天的动态状态。

默认只读取公开内容文件：

- `data/tweets.js`
- `data/note-tweet.js`
- `data/article.js`
- `data/profile.js`

私信、联系人、手机号、邮箱、IP、登录设备、令牌、应用授权和广告画像默认禁止读取，也不得进入公开案例或仓库。即使用户授权某项私密分析，也要按最小必要范围处理。

有检索脚本或已验证派生报告时优先用它们缩小范围，再回到原始记录核对。任何历史数字都写清“截至哪一天”。

## 公开 Skill 与私有资料的边界

公开 GitHub 可以包含：

- 通用方法、路由规则、模板、字段说明；
- 经过脱敏且得到授权的案例；
- 外部知识源的接入说明。

公开 GitHub 不应包含：

- 用户本地绝对路径、私信、客户资料和账号安全数据；
- 飞书内部页面全文、供应商信息和未公开商业数据；
- 会频繁变化的粉丝、收入、价格、城市、合作与业务状态。

合伙人、顾问、客户或社群成员提供的资料默认属于内部来源。没有明确公开授权时，只沉淀脱敏后的通用方法，不在公开仓库暴露原文链接、身份、聊天内容或具体经营数据。

## 没有工具或权限时

不要假装已经读过资料。告诉用户当前缺少哪一页或哪类证据，并给一个最短补充方式，例如：粘贴相关段落、导出一页、提供脱敏数据或授权只读访问。
