# 成交文案：从被看见到下一步

## 一、先判断该写什么

文案只解决表达、信任和行动问题，不能替代需求与产品。

| 当前卡点 | 应写的东西 | 不要写 |
| --- | --- | --- |
| 不知道用户是谁 | 访谈邀请、用户问题帖 | 品牌宣言 |
| 不确定需求 | 问题型内容、公开验证帖 | “市场巨大”的长文 |
| 交付说不清 | 一页最小报价 | 空泛观点 |
| 没人咨询 | 获客内容、主动私聊开场 | 只晒生活或鸡汤 |
| 有咨询不成交 | 需求复述、案例证据、报价、跟进 | 继续追曝光 |
| 交付亏钱 | 边界、验收、涨价说明 | 继续加量卖 |

如果真正卡点不是文案，明确告诉用户：“现在不该写文案。”然后给更靠前的动作。

## 二、先写清内容为谁服务

用一句话说明：

> 我为【具体人群】记录或解决【具体场景的问题】，让他下一步【评论 / 私聊 / 预约 / 试用 / 购买】，最终承接【商品或服务】。

没有商品时，可以承接访谈、候补名单、付费小测试或人工小样。不能永远“先做流量，以后再想”。

## 三、六块成交文案

不是每条都要机械写满六块，但缺少的部分必须是有意省略。

1. **自我识别**：第一句让目标用户知道“这和我有关”。
2. **真实场景**：写他正在做什么、卡在哪里，不写空泛人群标签。
3. **问题代价**：时间、金钱、机会、风险或情绪成本；不夸大焦虑。
4. **可信证据**：亲历、过程、样品、演示、用户授权案例或失败数据。
5. **具体方案**：做什么、多久、交付什么、不包含什么。
6. **唯一行动**：只留一个清楚、合规的下一步。

### 五行最小模板

```text
如果你正在【具体场景】，这条写给你。
大多数人卡住，不是因为【表面原因】，而是【真实原因】。
我最近用【真实过程/样品/经历】验证了【可核验结果或发现】。
现在我提供【交付物】，适合【对象】，不包含【边界】。
如果你想【下一步】，请【唯一行动】。
```

没有结果证据时，把第四行改成：“我正在验证这个方案，目前只有【已有事实】，所以先招募【数量】位做【有边界的小测试】。”数量必须真实，不能人为制造稀缺。

## 四、按赚钱链条选择文案

### 1. 获客帖

目标：让目标用户主动暴露自己。

结构：人群 + 场景 + 一个反常识判断 + 一个小方法 + 轻行动。

### 2. 证据帖

目标：降低不信任。

结构：起点 + 做了什么 + 真实结果 + 没解决什么 + 适用边界。失败或一般结果也可以成为证据，不包装成胜利。

### 3. 报价帖或报价页

目标：让用户能判断“适不适合、值不值”。

必须写明：对象、问题、交付物、周期、价格、数量、修改次数、验收和售后。

### 4. 私聊开场

目标：得到有效对话，不是立刻推销。

```text
看到你提到【具体问题】。我不确定自己理解得对不对：
你现在是用【现有做法】，但卡在【具体环节】吗？
如果方便，我想问两个问题，先判断我能不能帮上忙。
```

### 5. 需求复述与报价

目标：确认双方说的是同一件事。

```text
我理解你现在要解决的是【问题】，理想结果是【验收结果】，时间是【周期】。
我建议先做【最小交付】，包含【内容】，不包含【边界】，费用【价格】。
如果这些理解没偏，我们下一步是【付款/资料/会议】。
```

### 6. 未成交跟进

目标：拿到真实阻力，不纠缠。

```text
上次方案你还没有继续，我想确认一下：主要是需求暂时不急、方案不合适、预算、信任，还是决策时间的问题？
你直接说最真实的原因就行，我也好判断是否需要调整；如果现在不需要，我就先不打扰。
```

## 五、内容类型与商业作用

| 内容类型 | 作用 | 素材 | 下一步 |
| --- | --- | --- | --- |
| 问题型 | 让用户认出自己 | 用户原话、失败现场 | 评论场景或参加访谈 |
| 经历型 | 建立活人感 | 选择、过程、代价 | 关注实验或继续交流 |
| 方法型 | 展示解决能力 | 清单、演示、小样 | 试用第一步 |
| 证据型 | 降低风险 | 数据、前后对比、复盘 | 预约或报价 |
| 转化型 | 提出商品 | 对象、交付、价格、边界 | 购买或咨询 |

账号可以有生活和观点，但长期必须让目标用户知道：你在研究什么、做过什么、能提供什么、下一步怎么联系。

## 六、发布前检查

- 谁会停下来？
- 他为什么觉得“在说我”？
- 哪一句是事实，哪一句只是判断？
- 信任来自什么可核验证据？
- 看完只需要做哪一个动作？
- 这个动作连接赚钱链条的哪一环？
- 有没有夸大、虚假稀缺、隐私、侵权或平台风险？

至少输出一个可直接使用的完整版本，不只给标题和框架。

## 七、发布后只看商业数据

```text
曝光 -> 主页访问 -> 目标用户互动 -> 有效咨询
     -> 明确报价 -> 付款 -> 交付 -> 正毛利 -> 复购/转介绍
```

每周只优化最弱一层：

- 曝光低：查选题与开头。
- 曝光高、咨询少：查人群、证据和行动邀请。
- 咨询多、报价少：查提问和需求复述。
- 报价多、付款少：查需求、信任、价格与决策人。
- 付款后亏钱：查成本、范围、修改与售后。
