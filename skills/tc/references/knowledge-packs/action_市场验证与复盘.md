# 市场验证与复盘

## 这个知识包解决什么

用户已经知道大概要做什么，却一直学习、讨论、设计或换方向时，把想法变成一个损失可控的真实实验。

## 实验六件套

1. 要验证的唯一假设。
2. 触达哪一类真实用户。
3. 24 小时内看得见的交付物。
4. 七天只看一个指标。
5. 最多投入多少时间和钱。
6. 出现什么结果就继续、修改或停止。

## 相关原子

- `TC-ACT-001`：做 → 记录 → 复盘 → 再做。
- `TC-EXP-002`：实验前写渠道、价格、交付、周期、最大损失和两种后续。
- `TC-EXP-001`：同时记录触达、咨询、订单、转化、定制需求和交付成本。
- `TC-DECIDE-001`：低代价选择不要无限比较。
- `TC-LEVER-001`：低成本杠杆必须连接产物、用户或决策。

## 证据强弱

```text
朋友觉得不错 < 点赞收藏 < 目标用户原话 < 有效对话 < 明确报价
              < 支付定金 < 完成回款 < 正向毛利 < 复购或转介绍
```

## 复盘顺序

结果不好时，从商业链条最前面开始查。不要因为一条内容没流量就换赛道，也不要因为一次成交就急着招人、投流和自动化。
