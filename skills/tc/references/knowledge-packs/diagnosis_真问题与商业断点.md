# 真问题与商业断点

## 这个知识包解决什么

用户说了很多项目、情绪、机会和合作关系时，先找到最早断掉的一环，不急着给十个方案。

## 最小诊断

商业化问题优先借助核心参考源《技术合伙人｜商业世界观与 AI 实操框架》中“问题定义、最大未知、对标和单一变量”的视角；它只负责形成假设，不替代当前证据。

1. 分开事实、解释、情绪和假设。
2. 画出链条：目标用户 → 真实问题 → 商品 → 获客 → 有效对话 → 报价 → 付款 → 交付 → 毛利 → 复购。
3. 只选最早断掉的一环。
4. 把问题写成：在什么限制下，为了什么结果，现在要解决哪个唯一问题，用什么证据判断。

## 必问卡片

- `TC-USER-001`：谁会买、为什么现在买、结果被包装成什么商品？
- `TC-NEED-001`：是否省时间、降低成本、少走弯路或解决真实麻烦？
- `TC-PROD-001`：服务谁、解决什么、交付什么？
- `TC-MODEL-001`：现有卖家怎样获客、收费和交付？
- `TC-DECIDE-001`：方案差不多时，能否用更便宜、可逆的动作拿反馈？

## 输出标准

只交付一个真问题、一个当前判断和一个最能改变答案的问题。用户确认后再给方案。简单问题直接回答，不强行深挖。

## 边界

知识卡只帮助形成假设。最终以用户当前事实、用户原话、报价、付款、交付和毛利为准。
