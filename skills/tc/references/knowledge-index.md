# TC 知识包路由

只有当前问题确实需要 TC 历史方法或案例时才读取。先选一份核心参考源作为判断视角，再选一个最相关的知识包落到行动；不要一次加载全部。

## 核心参考源路由

| 当前问题 | 优先读取 |
| --- | --- |
| 商业化、需求、产品、流量、AI、To B 交付、创业取舍 | `core-sources/技术合伙人_商业世界观与AI实操.md` |
| 普通人起步、行动、内容、个人 IP、连接、社群、公开成长 | `core-sources/天策_公开实践与创业价值观.md` |

参考源负责帮助提问和判断，不负责证明当前事实。两份参考源冲突时，以用户当前说明、付款、交付、毛利和用户原话为准。

## 专项知识包路由

| 当前问题 | 读取知识包 |
| --- | --- |
| 想法很多、真问题不清、业务链断点不明 | `knowledge-packs/diagnosis_真问题与商业断点.md` |

| 技术不知道卖什么、需求和交付边界不清 | `knowledge-packs/product_需求产品化与交付.md` |
| 社群、活动、连接、合作和退出机制 | `knowledge-packs/community_组织与合作边界.md` |

需要查原始推文或解释某条知识来自哪里时，转到 `tc-knowledge`；主 `/tc` 不扫描 489 条原文。
