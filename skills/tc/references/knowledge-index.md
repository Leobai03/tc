# TC 知识包路由

只有当前问题确实需要 TC 通用方法时才读取。先选一份核心参考源作为判断视角，再选一个最相关的知识包落到行动；不要一次加载全部。

核心参考源、知识包和知识原子可以提供方法，但不得把作者或案例人物的经历映射成用户经历。主 `/tc` 不读取历史原推；历史原推只在用户明确查询作者过去表达时由 `tc-knowledge --scope posts` 返回，且不得反向参与项目推荐。

## 核心参考源路由

| 当前问题 | 优先读取 |
| --- | --- |
| 商业化、需求、产品、流量、AI、To B 交付、创业取舍 | `core-sources/技术合伙人_商业世界观与AI实操.md` |
| 普通人起步、行动、内容、个人 IP、连接、社群、公开成长 | `core-sources/天策_公开实践与创业价值观.md` |

参考源负责帮助提问和判断，不负责证明当前事实。两份参考源冲突时，以用户当前说明、付款、交付、毛利和用户原话为准。

## 外部理论库路由

| 当前问题 | 可选读取 |
| --- | --- |
| 用户明确要求结合 dontbesilent / DBS books / 1,544 条精选推文 | `external-sources/dbs-books.md`，再用 `tc-knowledge --scope dbs-books` 搜索 Markdown |

外部理论库不是第三份核心参考源。对话态只在用户明确要求、需要对照外部观点，或内部方法缺少相关案例时按需读取；每次只取 3 至 5 条相关记录，并保留日期、原链接与 CC BY-NC 4.0 非商业边界。

维护态可以把相关记录加工成本机 L1 研究候选，但候选不会自动进入本文件的路由，也不得仅靠改写升级。只有独立市场证据、反例检查和人工许可审查都通过后，才另行提炼为 TC 方法。原文不能变成商业文案或付费产品材料。
