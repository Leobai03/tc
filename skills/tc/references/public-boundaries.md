# 公开与商业边界

## 一、真实性

- 历史数字必须标注发生时间，不能当作当前状态。
- 营收、利润、流水、GMV 和回款必须分开。
- 案例未经授权必须匿名、删去可识别细节。
- 没验证的判断写成假设，不写成事实。
- 不制作虚假截图、评价、订单、倒计时和稀缺名额。

## 二、平台合规

- 不指导规避审核、谐音暗号、伪装导流或批量骚扰。
- 不刷粉、刷赞、刷评论或诱导虚假互动。
- 不蹲守同行评论区或直播间抢客，不用小号冒充中立用户建立虚假信任。
- 不利用平台漏洞、模拟器、批量账号或自动化行为操纵曝光和互动。
- 可研究对标的公开结构和用户问题，但不得搬运、洗稿、盗用账号人设或侵犯版权与商标。
- 发布前核对平台最新广告、导流、金融和 AI 内容规则。
- 主账号不测试明显高风险玩法。

## 三、交易合规

- 不承诺稳赚、保本、固定收益或收入结果。
- 不放大恐惧逼单，不制造虚假名额、倒计时、涨价或排队人数。
- 不使用“拉人头、层级返利、静态收益”等涉传销结构。
- 商品、价格、退款、发票、隐私和售后规则必须清楚。
- 涉及金融、证券、加密资产、税务和法律时，提示用户咨询持牌专业人士并核对当地规则。

## 四、隐私和安全

公开前移除：

- API Key、密码、Cookie、服务器和后台地址。
- 私人邀请链接、供应商线路、底价和结算比例。
- 客户手机号、微信、订单、聊天和公司内部数据。
- 未授权的人脸、姓名和合作争议细节。

## 五、禁止业务

不帮助设计或推广：

- 学术代写、考试作弊和证件造假。
- 欺诈、侵权、盗版、恶意爬取和账号攻击。
- 操纵、骚扰、胁迫或利用弱势群体的成交方式。
- 法律或平台明确禁止的商品与服务。

## 六、公开 Skill 的边界

公开的是方法和判断过程，不是商业秘密仓库。

可以公开：

- 需求验证方法。
- 起号和内容结构。
- 失败案例的抽象教训。
- 报价、交付和复盘模板。
- AI 协作方法。

不公开：

- 当前客户、供应商和合作关系。
- 真实后台、接口、专线、邀请码和价格底牌。
- 未经同意的原始聊天、合同和会议内容。
- 可以直接伤害现有用户或合作方的信息。

## 七、合作与组织边界

- 不用故意制造冲突、诱骗或监控测试员工和合作方。
- 不用“制衡”替代授权、访问控制、审计和透明的岗位责任。
- 重要合作必须写明出资、分工、决策权、知识产权、客户归属、结算、违约和退出。
- 合同不能替代对合作方资源、履历、交付能力和利益冲突的核验。
