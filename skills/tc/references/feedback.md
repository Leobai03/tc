# Skill 问题反馈闭环

## 触发信号

用户表达以下意思时启用本流程：

- “这个 Skill 有问题 / 不好用 / 判断错了。”
- “这段回答太长、太空、做不了。”
- “它漏掉了一个重要环节。”
- “我希望增加一个功能。”
- “帮我把这个问题反馈给作者。”

不要因为用户普通地反对一条商业建议，就自动判定 Skill 有缺陷。先判断是方法问题、信息不足、执行结果不同，还是确实存在可复现的 Skill 问题。

## 隐私事实

- Skill 不会自动收集或上传全部使用者对话。
- 作者默认看不到用户与 AI 的聊天记录。
- 只有用户明确选择反馈并授权发送时，才能把反馈发往作者邮箱。
- 不得默认附上整段对话，只提取复现问题所需的最少内容。

## 整理反馈

优先在当前上下文中填写，缺少但不影响理解的信息不要追问：

```text
主题：一句话描述问题
使用场景：用户原本想完成什么
实际表现：Skill 做了什么
期望表现：用户希望它怎么做
影响：为什么这会妨碍使用
复现步骤：如何再次遇到该问题
建议：用户的改进想法（可选）
环境：使用平台、模型或 Skill 版本（知道才写）
```

保留能帮助复现的用户原话，但不要附上无关聊天。

## 发送前脱敏

移除或泛化：真实姓名、手机号、身份证、住址、客户资料、未公开营收、合同、聊天对象、账号密码、Cookie、API Key、后台地址、私人链接、供应商底价及其他商业秘密。

如反馈必须包含敏感内容才能复现，明确告诉用户具体要发送什么，并在发送前单独取得同意。

## 发送规则

1. 先向用户展示整理后的收件人、主题和正文。
2. 明确说明不会自动附上完整对话。
3. 只有用户明确同意发送后，才调用当前可用的邮件工具发送到 `1179884054@qq.com`。
4. 如果没有可用邮件工具，提供可复制的邮件正文和 [点击写邮件](mailto:1179884054@qq.com)，不得声称已经同步或发送。
5. 发送成功后，只确认收件地址和主题，不重复公开敏感正文。
6. 用户只想口头吐槽、不想发送时，继续解决当前问题，不施压索取反馈。
