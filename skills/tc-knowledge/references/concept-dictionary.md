# TC 高频概念词典

这份词典用来防止同一个词在不同对话里被说成不同意思。遇到争议时，先回到可观察事实，不用术语压人。

| 概念 | TC 中的意思 | 不等于 |
| --- | --- | --- |
| 事实 | 已发生、可核对的动作或数据 | 感觉、预测和别人应该怎样 |
| 解释 | 人对事实的理解 | 事实本身 |
| 假设 | 尚未被市场证明、可以设计实验验证的判断 | 结论 |
| 真问题 | 当前最限制现实结果、解决后会改变下一步的问题 | 最让人焦虑的表面话题 |
| 目标用户 | 正在经历具体问题、能被找到且可能决定付款的人 | “所有创业者”“所有企业” |
| 用户原话 | 目标用户在真实场景里的原始表达 | AI 帮用户总结出来的痛点 |
| 有效对话 | 对方符合目标用户，并具体谈到场景、代价、现有方案或决策条件 | 点赞、寒暄和泛泛说“挺好” |
| 最小可售卖方案 | 能说明对象、结果、范围、价格、交付和下一步的一页商品 | 做完的完整平台 |
| 报价 | 向真实客户给出明确范围、价格和成交条件 | 问朋友“这个值多少钱” |
| 流水 | 一段时间内经过账户的金额 | 营收、回款、利润或个人收入 |
| 营收 | 已确认向客户销售商品或服务的金额，需说明口径 | 账户余额和利润 |
| 回款 | 已经实际收到的钱 | 签约额、口头承诺和应收款 |
| 毛利 | 回款减去直接交付成本 | 净利润和可自由支配现金 |
| 正向单位经济 | 每完成一单，扣除直接获客和交付成本后仍有正贡献 | 总营收看起来很大 |
| 市场验证 | 用访谈、报价、付款、交付、复购或明确拒绝减少不确定性 | 发一条内容后有人点赞 |
| 停止条件 | 实验开始前写明，出现哪些结果就停、改或降级 | 失败后的自我安慰 |
| 个人 IP | 别人对“你长期解决什么问题、用什么方式做事”的稳定认知 | 永远正确的成功人设 |
| 社群 | 围绕一组持续问题提供筛选、反馈、协作或陪伴的产品 | 把很多人拉进同一个群 |
| 产品化 | 把能力整理成对象、结果、边界、流程、价格和验收可重复的商品 | 做一张海报或起一个名字 |
| 数字员工 | 有职责、权限、工具、状态、反馈和失败处理的自动化岗位 | 只接上一个大模型聊天框 |
