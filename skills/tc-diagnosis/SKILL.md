# TC Diagnosis｜创业问题消解

## 目标

把“我不知道怎么办”或说出口的表面问题，改写成用户真正想问、真正需要解决，并且能够被行动和数据验证的问题。一次只确定最早、最限制结果的一环。

## 工作流

1. 从对话中提取四类信息：
   - 事实：已经发生且能核对的动作与数据。
   - 解释：用户对事实的理解。
   - 情绪：焦虑、兴奋、委屈、羞耻或怕错过。
   - 假设：尚未被用户、付款或交付证明的判断。
2. 按顺序检查赚钱链条：

   ```text
   目标用户 -> 真实问题 -> 可售卖方案 -> 获客 -> 有效对话
           -> 报价 -> 付款 -> 交付 -> 毛利 -> 复购/转介绍
   ```

3. 找到最早断掉的一环。前一环未通时，不优化后一环。
4. 判断用户处于哪个阶段：找人群、验需求、做商品、获客、成交、交付或复现。
5. 信息不足且会改变判断时，一次最多问三个问题；能够用低成本实验补证据时，直接给实验。
6. 对照用户想要的现实结果，明确写出“表面问题”和“真正问题”的区别；两者本来一致时直接确认，不强行制造深层问题。

## 输出

```text
一句话结论：
现在不是【表面问题】，而是【最早断掉的业务环节】。

你真正想问 / 真正需要解决的是：

已知事实：
-

仍是假设：
-

唯一卡点：

现在先不做：

需要补的证据：
```

不要用空泛安慰替代判断，不要把粉丝、营收、流水和利润混为一谈。涉及动态平台规则、投资、法律、税务或医疗时，先核对最新官方信息。
