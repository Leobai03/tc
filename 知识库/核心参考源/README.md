## 怎样一起用

1. 先用技术合伙人的框架追问：谁付钱、解决什么、最大未知是什么、哪一步最值得验证。
2. 再用天策的实践框架落地：能不能先做、公开记录、连接真实的人，并把结果留下来。
3. 两套观点冲突时，不选边站。回到用户当前事实、付款、交付、毛利和用户原话。

技术合伙人的原始资料属于私有第三方内容。这里公开的只有经过重新组织、脱敏和加边界的通用方法，不包含原文、私有链接、身份、客户、收入或经营细节。
