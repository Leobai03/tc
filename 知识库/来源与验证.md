# 来源与验证规则

## 来源层级

| 等级 | 来源 | 使用方式 |
| --- | --- | --- |
| A | 真实付款、退款、交付、复购、毛利与用户原话 | 可以支持具体业务判断，仍需说明时间和样本 |
| B | 作者亲自执行后的过程记录与复盘 | 作为经验使用，不自动推广到所有行业 |
| C | 公开项目、公开内容、行业案例和第三方方法 | 保留来源，只吸收通用方法，关键事实另行核验 |
| D | 个人观点、推测、情绪和未经核验的数据 | 只能作为待验证假设，不写成事实 |

## 当前外部参考

- [dontbesilent2025/dbskill](https://github.com/dontbesilent2025/dbskill)：项目结构用于可选协同和架构研究，不宣称官方合作。`books/` 中的开源推文集作为可选外部理论库，只从原作者仓库同步 Markdown 到用户本机，按 CC BY-NC 4.0 署名、非商业使用；不打入 TC 的 Apache-2.0 安装包，也不把原文改写成商业内容。
- [jycakfi-web/-skill](https://github.com/jycakfi-web/-skill)：只吸收生存优先、调研、资源匹配、对标、小成本试错、书面分工和带数据复盘等通用方法。其人物身份、经营数字、故事与评价不作为 TC 的已证实事实。
- 天策的公开内容与个人实践：只收录能说明来源、时间、动作和结果的部分。个人观点不因作者身份自动变成普遍规律。
- 技术合伙人的知识付费与实操记录：作为 TC 商业化、AI、产品、流量与 To B 交付的核心参考源。公开仓库只保留脱敏、重写并加边界的通用框架，不转载原文、私有链接、身份、客户、收入或动态工具结论。涉及绝对化商业判断、操控性做法和无法核验的数据时降级或剔除。核心参考源是“高频使用的判断视角”，不是“更高等级的事实证据”。

## 外部知识源优先级

同一用户同时提供当前对话、飞书知识库、本地内容归档和派生报告时：

1. 当前对话中明确更新的事实优先。
2. 带更新时间的当前状态页优先于历史档案。
3. 原始记录优先于派生总结；派生总结负责缩小范围，不取代原始证据。
4. 旧内容、个人观点和第三方材料只作历史、经验或待验证假设。

外部知识源的接入与隐私边界见 [`外部知识源接入.md`](外部知识源接入.md)。

## 动态事实

以下信息使用时必须读取最新资料或重新核对：

- 平台规则、模型能力、产品价格和法律政策；
- 作者及案例主体的粉丝、收入、城市、年龄、团队和业务状态；
- 合作关系、分成、合同、供应商、客户和项目是否仍在运营。

## 进入知识库的最低条件

一条新方法至少回答：

1. 它解决哪个具体问题？
2. 来源是什么，发生在什么时候？
3. 做了什么动作？
4. 得到了什么正面或负面证据？
5. 适用边界和停止条件是什么？
6. 是否包含隐私、商业机密或未经授权的信息？
7. 什么反例会推翻它，出现什么副作用应降级？
8. 来源许可证是否允许公开和商业使用？

无法回答时，先留在待验证区，不进入正式知识包。

## 用户市场证据反馈

`tc-state` 中的市场证据默认只属于用户本机项目。只有用户针对当次结果明确同意后，才可以导出一份去除项目名、联系人、完整聊天、客户与凭据的本机候选，再交给 `tc-knowledge`。

导出和加入候选区都不代表公开授权，也不代表方法有效。单个案例不能升级 L2/L3；多个独立案例出现相近场景、动作和结果后，仍需人工检查反例、隐私、许可与选择偏差。

## 证据与许可分离

- L1/L2/L3 表示方法证据状态，不表示版权或商业许可状态。
- CC BY-NC、私有资料或其他受限来源即使被验证，也不能自动进入商业输出。
- 公开 TC 方法必须有可公开的独立证据，并通过人工许可审查；不得用“独立改写”掩盖对第三方独特表达、结构或案例的复制。

大量外部资料的拆分、升降级和公开边界见 [`skills/tc/references/source-distillation.md`](../skills/tc/references/source-distillation.md)。

DBS 推文集的接入、检索和许可边界见 [`外部理论库/dbs-books.md`](外部理论库/dbs-books.md)。
