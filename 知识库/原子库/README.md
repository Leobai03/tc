# TC 公开知识原子库

这里保存的是少量、可追溯的公开知识卡片，不是把作者所有推文一股脑塞进 Skill。

当前共有 **22 条**原子，来源是 `@Leobai825` 的公开 X 历史内容，日期范围为 2025-12-16 至 2026-07-13。每条都带原推链接、发布日期、对象类型、证据等级、知识等级、可信度和适用边界。

## 小学生版解释

一篇推文像一大块面包，知识原子就是切成一片一片：

- 一片只讲一个能拿来做事的意思；
- 背面写清楚从哪里切下来的；
- 过时的话、夸张的话和危险做法不能混进主食里；
- 历史推文只能证明作者当时这样表达，不代表今天的业务状态。

## 字段

| 字段 | 含义 |
| --- | --- |
| `id` | 原子的唯一编号 |
| `knowledge` | TC 可以采用或提醒用户验证的通用结论 |
| `original` | 公开原文的简短摘要，不代替原链接 |
| `url` | 原始公开推文链接 |
| `date` | 原文发布日期 |
| `topics` | 这张卡片属于哪些主题 |
| `skills` | 哪些 TC Skill 可能读取它 |
| `type` | 原则、案例、方法或边界提醒 |
| `object_type` | `QST / CON / OPI / CAS / SOL` 五类知识对象 |
| `level` | `L0` 原始资料、`L1` 候选、`L2` 已验证模式、`L3` 稳定方法 |
| `evidence_grade` | `A / B / C / D` 证据等级 |
| `confidence` | 当前证据可信度，不等于永远正确 |
| `status` | `historical-public` 表示历史公开表达 |
| `visibility` | 是否允许进入公开安装包 |
| `source_id` | 对应的来源登记编号 |
| `boundary` | 使用时必须一起看到的限制 |

## 使用规则

1. 先按当前问题检索主题，不要每次读取全部原子。
2. 原子只帮助提出假设和设计验证，不替代当前用户的真实数据。
3. 案例里的粉丝、收入、订单和转化只能按原日期引用，不能当今天的数据。
4. 带绝对化结论或高风险做法的原文，只能提炼成边界提醒，不能升级为 TC 原则。
5. 新原子进入这里前，必须通过仓库自动检查并人工核对原链接。
6. `L1` 只能启发实验；`L2` 可以在相似场景使用；只有跨案例稳定且边界清楚的内容才可能升级为 `L3`。
