# TC 知识库

知识库只是 TC 的书架，真正的产品是把混乱变成真问题和可验证动作的判断能力。TC 知识库不是创业百科或项目库，也不是一堆文章，而是一条可以检查的流水线：

```text
公开原推 / 私有飞书 / 外部资料
          ↓ 标记来源、日期和权限
       核心参考源
          ↓ 提供世界观与判断视角
       知识原子
          ↓ 验证、脱敏、补边界
       专项知识包
          ↓ 按当前问题只读一份
       /tc
```

受限外部理论另走一条维护链：

```text
DBS 原文（本机、非商业）
  → 3 至 5 条相关记录
  → L1 本机研究候选
  → 独立用户/付款/交付证据
  → 人工许可与反例审查
  → 才能另行判断是否形成 TC 方法
```

## 小学生版

- `公开内容索引/`：原书，记录过去公开说过什么；
- `核心参考源/`：两副看问题的眼镜，决定先追问什么，不替代事实；
- `原子库/`：从原书切出来、背面写着出处和注意事项的小卡片；
- `Skill知识包/`：把同一问题的卡片装成一本薄教材；
- `外部理论库/`：其他作者的书架卡，只在需要时回原仓库读，不能当成自己的书；
- `SOURCE_OF_TRUTH.md`：图书馆地图，告诉 AI 去哪一层找；
- `高频概念词典.md`：统一“营收、回款、毛利、验证”等词的意思。

## 三层权限

1. **公开层**：本人公开推文、通用方法、模板和脱敏授权案例，可以进入 GitHub。
2. **内部层**：飞书当前状态、实验、人工判断和合伙人资料，只在授权任务中读取。
3. **禁止公开层**：私信、客户、供应商、分成、账号安全数据和未经授权第三方原文。

技术合伙人的知识付费与实操记录已被提升为 TC 的核心参考源 A，主要负责商业化、需求、产品、流量、AI 和企业交付的判断视角；天策公开实践是核心参考源 B，主要负责行动、内容、个人 IP、连接和社群。技术合伙人的私有原文仍不进入公开仓库，公开的只是脱敏、重写并加过边界的通用框架。

  --output 知识库/公开内容索引/posts.jsonl --handle Leobai825

python3 tools/check.py

## 维护纪律

1. 历史原文只证明作者当时公开表达；不覆盖用户当前事实，也不参与用户的项目生成和方向推荐。
2. 新来源先登记日期、权限和用途；收录不等于认可。
3. 一次优先提炼 5 至 10 条高价值原子，不追求虚假数量。
4. 原子必须能说明适用场景、最小动作、有效证据和停止条件。
5. 平台、价格、模型、政策、合作和业务状态使用前重新核对。
6. 公开前完成脱敏、来源、日期、证据和授权检查。
7. 安装包内的知识副本由脚本生成，不直接手改。
8. 外部受限来源的“验证通过”和“允许公开/商用”分开判断；任何一级知识都不能自动改变上游许可。
