# 个人 IP 与内容获客

## 这个知识包解决什么

用户不知道发什么、感觉人设太高、内容和业务断开，或者只有流量没有下一步时使用。

## 内容先服务一个业务动作

每条内容只选择一个下一步：获得反馈、邀请访谈、展示过程、证明交付、引导预约或发出报价。没有业务动作时，也可以真实记录成长，但不要假装它已经完成变现。

## 相关原子

- `TC-ID-001`：成长型 IP 可以记录成功和失败，不必扮演导师。
- `TC-TRUST-001`：真实过程比成功包装更适合长期积累信任。
- `TC-CONTENT-001`：先做一件小事，再写问题、过程、结果和下一步。
- `TC-FDB-001`：公开表达的价值是获得纠偏反馈。
- `TC-ECOSYS-001`：给别人曝光可以成为生态实验，但必须核验成绩和保护隐私。

## 一条可用内容的五段

1. 一个目标读者能认出的具体场景。
2. 你刚做了什么，而不是只讲价值观。
3. 真实结果或仍未解决的问题。
4. 你现在怎样理解它，以及证据边界。
5. 一个清楚的下一步。

## 红线

不编收入、客户、名额、倒计时和评价；不为了人设隐瞒关键事实；不把流量当成付款和交付证据。
